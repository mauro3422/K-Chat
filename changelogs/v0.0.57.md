# Release Notes v0.0.57 (2026-06-14)

Esta versión rompe el ciclo de dependencia `tools ↔ context` que violaba el principio Lego de capas hermanas independientes, y suma nuevas herramientas de análisis arquitectónico con validación post-hook.

---

## Cambios Clave

### 🧱 Arquitectura Lego: Ciclo `tools ↔ context` eliminado
- **Problema**: `tools/save_memory.py` importaba `context.runtime.invalidate_context_cache` y `context/runtime.py` importaba `tools.get_default_registry` — un ciclo entre capas hermanas.
- **Solución**: Core (capa de orquestación) ahora inyecta ambas dependencias vía DI. `ToolExecutionService` (en `src/core/services/`) importa `invalidate_context_cache` y lo inyecta como callback al runner de tools. El system prompt recibe `tool_definitions` desde el tool registry vía `orchestrator → history_service → builder → runtime`.
- **Impacto**: 0 violaciones de upward coupling. Las capas `tools/` y `context/` ya no se importan mutuamente.
- **Archivos**: `save_memory.py`, `runner.py`, `runtime.py`, `builder.py`, `history_service.py`, `tool_execution_service.py`, `orchestrator.py`

### 🔍 Nuevas Tools de Análisis Arquitectónico
- **`dependency_graph`**: Grafo de dependencias entre módulos Python con clasificación por dirección (downward, same-layer, upward, banned).
- **`find_dead_code`**: Detecta código no referenciado: funciones/clases definidas pero nunca importadas fuera de su archivo.
- **`impact_analysis`**: Analiza el impacto de cambiar una función/clase: encuentra llamadores directos e indirectos.
- **`_arch_checker`**: Helper interno que verifica reglas arquitectónicas (upward coupling, framework imports en domain layers).
- **`_preflight`**: Helper interno que valida operaciones de archivo antes de aplicarlas (pre/post flight checks).

### 📊 Complejidad Ciclomática en `analyze_code`
- Nueva métrica de complejidad McCabe por función.
- Parámetro `complexity` (default: True) para calcular y flaggear funciones que exceden umbrales.
- Labels visuales: Baja (≤5), Moderada (6-10), Alta (11-20), Muy Alta (21-40), Inextricable (>40).

### 🛡️ Post-hooks de Arquitectura en Tools de Edición
- **`edit_file`**: Nuevos parámetros `arch_check` y `verbose` que ejecutan validación arquitectónica + impact analysis post-edición.
- **`write_file`**: Post-flight validation que verifica reglas arquitectónicas después de escribir.
- **`validate_all`**: Nuevo modo `architecture=True` que verifica reglas Legos ademas de sintaxis.

### ⚡ Model Availability Checker
- `web/static/model-availability.js`: Script inline externalizado que muestra dots de estado de modelos en vivo.

### 🧪 Tests
- 73 tests unitarios pasando en los módulos afectados por el refactor del ciclo.
- Tests de `test_runtime.py`, `test_runner.py`, `test_orchestrator.py` actualizados para reflejar la nueva DI.
