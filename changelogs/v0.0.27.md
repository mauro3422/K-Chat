# v0.0.27 — Database lifecycle guard and facade cleanup

## Bug Fixes

### Database lifecycle
- Separated raw connection reuse from schema initialization in `src/memory/database.py`.
- `get_conn()` now reuses the thread-local connection and only triggers initialization once per DB path.
- `init_db()` remains the explicit migration entry point and marks the path as initialized after success.

### API facade
- Removed the unused `get_verified_models()` export from `src.api.chat`.
- Kept the API facade narrower so the model-verification seam stays in `src.llm`.

### LLM package surface
- Removed `get_verified_models()` from `src.llm.__init__` so package-level imports stay focused on chat and provider registration.
- `get_default_model()` now prefers the cached verified-model list before falling back to a fresh free-model scan, and the cached list overrides the old `big-pickle` special-case.

### Frontend compatibility
- Replaced the debug panel's inline `onclick="toggleDebug()"` hooks with DOM listeners, removed the global `toggleDebug` alias, and wired `session.js` to the module export directly.

## Tests

- `tests/unit/test_database.py`

## Notes

- This is a smaller cut than the repo-registry removal, but it makes the DB lifecycle and model seam more explicit without changing external behavior.
