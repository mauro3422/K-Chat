# v0.0.52 — 2026-06-12

## Mega-refactor: Lego Architecture Consolidation

### Model State & LLM Layer
- **ModelState class**: Encapsulated all global mutable state (failed/verified/cached models) into injectable `ModelState` class with thread-safe methods
- **`switch_model()` fix**: Never returns a failed model; raises `RuntimeError` when all models are exhausted
- **Policy split**: Divided `policy.py` into 4 focused sub-modules — `discovery.py`, `verifier.py`, `selector.py`, `failover.py`; `policy.py` remains as backward-compat facade
- **Wildcard imports removed**: `models.py` now uses explicit imports from `providers` and `model_state`
- **Backward compat**: Module-level aliases (`PRIORITY`, `is_model_failed`, etc.) bound to `_DEFAULT_STATE` instance

### Repository Injection & Decoupling
- **Repos injection**: `tool_loop.py` now accepts `Repositories` dataclass instead of directly instantiating `MessageRepository()`; `orchestrator.py` passes repos through; `routers/chat.py` creates repos via `get_repos()`
- **`chat_sync.py` delegation**: Rewritten to delegate to `orchestrator.chat_stream()` instead of duplicating initialization logic; zero production callers impacted
- **`_tool_persister.py`**: Verified already using public `record_execution()` API (no raw SQL); docs updated

### Data Access Layer
- **`sqlite3.Row` row_factory**: All DB rows support named column access (`row["role"]` instead of `row[0]`)
- **Positional access migration**: `history_parser.py`, `history_ui.py`, `message_renderer.py` migrated to named access with graceful fallback for plain tuples (tests)

### Retry & Stream Resilience
- **Save retry**: `_save_with_retry` upgraded to 3 attempts with exponential backoff (1s, 2s)
- **Mid-stream recovery**: `build_stream_generator` now attempts `StreamRetryHandler` recovery on transport errors when partial content exists
- **`StreamRetryHandler`**: `DEFAULT_MAX_RETRIES` 1→2; wireado en `routers/chat.py`
- **Rate limiter**: Fixed race condition (`del _session_rate` outside lock) — dead code removed
- **Critical tests added**: `test_model_state.py` (6 tests), `test_save_retry.py` (2 tests), stream retry handler tests expanded

### Frontend Cleanup
- **CSS extraction**: 78 inline `el.style.X = Y` assignments in widget toolbar/editor/history/iframe moved to CSS classes in `style.css`
- **Global removal**: `window.KairosWidgets`, `window.loadSession`, `window.KairosSession` window bridges removed; modules import directly
- **`logUI`/`logStream` extraction**: Extracted to standalone `log-ui.js` module with zero dependencies; 17 modules migrated to direct imports; `debug.js` updated to use same shared state
- **`shared-state.js`**: Created for centralized get/set state without global leakage
- **`defaultModel` fix**: `chat-form.js` uses `getDefaultModel()` with try/catch for test environments
- **Bootstrap cleanup**: `chat-form-bootstrap.js` and `stream-bootstrap.js` window assignments removed

### New Tools & Features
- **`git_operation` tool**: Safe Git operations (status, diff, log, branch, add, commit, push, pull, clone); blocks destructive commands (`--force`, `--hard`); auto-discovered (16 tools total)
- **`MemoryIndexRepository`**: New repo for `memory_index` table with upsert/get/get_all/delete; integrated into `get_repos()` and migration 011 cleanup

### Documentation
- **Docs updated**: README.md (v0.0.51, 15 tools), ARCHITECTURE.md, ARCHITECTURE_SUMMARY.md, ARCHITECTURE_FRONTEND.md (15→26 modules), MODULES.md, AGENTS.md (10→15 tools)
- **`AGENTS.md`**: Tool list updated to include `search_files`, `edit_file`, `analyze_code`, `list_files`, `execute_command`
