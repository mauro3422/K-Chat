# v0.0.63 — Reranker ONNX + Notificaciones + Retrieval sin auto-referencia

> Fecha: 2026-06-17
> Tags: `backend` `frontend` `memoria` `retrieval` `notificaciones` `bugfix`
> Tests: `npx vitest run --config vitest.ts.config.js` + `pytest tests/unit/test_stream_contract.py`

## Retrieval fixes (sesión actual + MEMORY.md duplicado)

- **Filtro de sesión actual**: `VectorStore.search()` ahora acepta `exclude_source_key`; `HybridRetriever` lo pasa como `session_id`; `RetrievalService` envía `source_filter='session'` para excluir `source='memory'` (ya incluido en MEMORY.md)
- **5 archivos modificados**: `store.py`, `keyword_search.py`, `entity_search.py`, `hybrid_retriever.py`, `retrieval_service.py`

## Reranker migrado a fastembed (ONNX, sin torch)

- **`reranker.py`**: Reemplazado `sentence_transformers.CrossEncoder` (PyTorch, ~700MB) por `fastembed.rerank.cross_encoder.TextCrossEncoder` (ONNX, ~10MB)
- **Modelo**: `cross-encoder/ms-marco-MiniLM-L-6-v2` → `Xenova/ms-marco-MiniLM-L-6-v2` (mismos pesos, export ONNX)
- **Rendimiento**: ~4-5ms por inferencia vs ~20ms antes (CPU)
- **`requirements.txt`**: `fastembed>=0.5.0` → `>=0.8.0` (requerido para `TextCrossEncoder`)

## Sistema de notificaciones Python → Frontend

- **Nuevo evento `"notification"`** en protocolo NDJSON (`stream_contract.py`) y SSE (`SSEClient.ts`, `sse.ts`)
- **`StreamOrchestrator.ts`**: Handler que parsea `{type, message, duration}` y emite `notification:show` al EventBus
- **`SSEClient.ts`**: Nuevo handler `handleStreamNotification()` para notificaciones SSE
- **`orchestrator.py`**: Emite notificación `"warning"` cuando el reranker no está disponible
- **`retrieval_service.py`**: Retorna tupla `(memory_block, degraded)` propagando el flag

## Contador de exchanges corregido (frontend)

- **`SessionStore.addMessage()`**: Ahora solo incrementa `session.count` para `role === 'user'` — tool_calls y tool_results ya no inflan el contador durante la sesión en vivo

## Recencia de memorias activada

- **`store.py:search()`**: Después de cada búsqueda, ejecuta `UPDATE vec_meta SET query_count += 1, last_accessed = ?` para los rowids retornados
- Columnas que antes nunca se actualizaban ahora reflejan uso real

## Inconsistencias corregidas

- **`vectorize_sessions.py`**: Texto guardado en `vec_meta` ahora usa `[:4000]` (igual que el embedding) en vez de `[:2000]` — eliminada la inconsistencia
- **`migrations.py`**: Nueva migración `023` que agrega `UNIQUE INDEX (session_id, key)` a `memory_index` — `ON CONFLICT` ahora funciona correctamente

## Debug panel

- **`debug.py`**: BACKEND LOGS ya no devuelve siempre vacío — ahora lee directamente de `logs/server/*.jsonl`

## Sesiones sidebar

- **`session_repository.py`**: Contador cambiado de `COUNT(*)` (mensajes totales incluyendo tool_calls/results) a `SUM(CASE WHEN role='user')` (exchanges reales)
- **`SessionList.ts` + `sidebar.html`**: Label cambiado de `"N msgs"` a `"N exchanges"`

## Archivos huérfanos eliminados
