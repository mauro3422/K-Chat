# Release Notes v0.0.56 (2026-06-14)

Esta versión introduce la refactorización modular de skills (estructura de piezas Lego) con cargadores y autodescubrimiento desacoplado, así como un entorno de desarrollo más inteligente y rápido basado en `pytest-testmon` para ejecución de tests selectivos.

---

## Cambios Clave

### 🧩 Refactorización Modular de Skills (Estructura Lego)
- **Subdirectorios Autocontenidos**: Agrupación de especificaciones de habilidades en la carpeta `skills/` en paquetes lógicos como `skills/db-query/` y `skills/html-widgets/`.
- **Reubicación de Herramientas**: Migración del script de base de datos SQL desde `src/tools/db_query.py` a `skills/db-query/tool.py`, convirtiéndose en una parte integral y autocontenida de su respectivo módulo de skill.
- **Herramienta `read_skill`**: Actualización de la lógica en `read_skill.py` para consultar directamente al nuevo registro dinámico `SkillRegistry` para servir e indexar las instrucciones en markdown.

### ⚙️ Cargador Desacoplado y Autodescubrimiento (Registry)
- **Autodescubrimiento**: Diseño e implementación de `SkillRegistry.discover_tools()` en `src/skills/registry.py` para automatizar y encapsular la importación dinámica (`importlib`) de herramientas de skills.
- **Abstracción Limpia**: Refactorización de `ToolRegistry.build()` en `src/tools/registry.py` para consumir la lista de herramientas directamente desde la interfaz del registro de skills, logrando un desacoplamiento completo a nivel de sistema de ficheros.
- **Indexación Dinámica**: Generación automática del archivo `skills/INDEX.md` en el arranque del servidor FastAPI (`lifespan`) para mantener sincronizado el catálogo.

### 🧪 Tests Inteligentes y Ejecución Selectiva (`pytest-testmon`)
- **Integración de `pytest-testmon`**: Instalación del plugin de ejecución selectiva inteligente y actualización de `requirements.txt`.
- **Rendimiento Celeron N4020**: Permite saltar la ejecución de los 807 tests completos y correr únicamente aquellos directamente afectados por cambios en el código de forma instantánea (0.24 segundos cuando no hay cambios).
- **Alineación de Suite**: Nuevos unit tests en `tests/unit/test_skills_registry.py` para verificar la robustez del cargador y resolución de tests unitarios del router y herramientas del sistema de skills.
