# v0.0.42

## Changes

- Rate-limit interruptions now include provider reset hints when the upstream response exposes `retry-after` or `x-ratelimit-reset-*` headers, and the UI shows the interruption reason instead of a generic failure.
- The model selector now reads the local OpenCode registry metadata and displays context, output size, modality, reasoning, and tool-call capability labels.
- Empty responses no longer auto-retry after the model has already produced reasoning or tool activity, which avoids retry loops that amplify cutoffs.
- `config.py` now accepts `OPENCODE_ZEN_API_KEY_FALLBACK` before failing startup, so a secondary OpenCode key can be used as a fallback.
