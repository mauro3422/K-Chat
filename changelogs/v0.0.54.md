# v0.0.54 — 2026-06-13

## Lego Hardening de Backend y Transición Frontend

### Backend / Core
- **Runtime sin wrappers viejos**: eliminados `src/api/llm.py`, `src/api/models.py`, `src/api/history.py` y `src/api/health.py`
- **`save_message()` legacy fuera del runtime**: `src/api/messages.py` quedó con `save_message_record()` como contrato canónico
- **Historial tipado**: `HistoryMessage` estabiliza el contrato entre parser, UI y reconstrucción
- **`rebuild_history()` explícito**: ahora exige `messages_repo` en vez de caer a `get_repos()` por defecto
- **Sesión más limpia**: la cascada de borrado se movió a `SessionRepository.delete_cascade()`
- **Delete 100% explícito**: `delete_session()` y `delete_cascade()` ya requieren `repos` inyectado, sin fallback a `get_repos()`
- **Contrato de sesión afinado**: `conn_fn` salió del contrato público de sesión

### Contexto / Tools
- **`load_context()` puro**: ya no escribe `TOOLS.md` ni archivos de reglas durante la lectura
- **Snapshot explícito**: la materialización de contexto quedó separada del armado del prompt
- **Loader más predecible**: `src.tools` ya no dispara el auto-build al importar el paquete

### Web / Frontend
- **Imports directos**: routers y servicios web dejaron de depender de wrappers obsoletos
- **Transición acotada**: la compatibilidad legacy quedó concentrada en las superficies que todavía la necesitan
- **Historia compartida más estable**: el frontend ya consume el contrato de historial normalizado
- **Session page hardening**: las acciones de sesión ahora se renderizan con DOM APIs y la navegación queda detrás de una dependencia explícita
- **Session page HTML**: el render principal ya parsea fragmentos DOM y el cancel de borrado restaura un clon del nodo
- **Session delete DOM**: el cancel de borrado ya restaura un clon del nodo, no un `outerHTML`
- **Bootstraps con nav**: `app.js` inyecta navegación explícita en `session-page.js` y `chat-form.js`
- **Render helpers**: `markdown-renderer.js`, `sidebar-refresh.js` y `content-handler.js` ya pintan con fragmentos DOM en lugar de asignar `innerHTML`
- **Widget builder**: `iframe-builder.js` ya monta loading/error con nodos DOM
- **Debug panel hardening**: los listados de UI, stream, widgets y backend logs dejaron de reconstruirse con `innerHTML`
- **Debug panel core render**: `refreshDebug()` pasó a construir el panel con nodos DOM explícitos en vez de concatenación HTML
- **Sidebar refresh shared**: la recarga del sidebar quedó centralizada en `sidebar-refresh.js` y consumida por sesión, stream y lifecycle
- **Debug listeners injectable**: `bindDebugControls()` puede recibir un `eventTarget`, quitando la dependencia directa de `window`
- **Stream completion helper**: el post-success del stream quedó aislado en `stream-completion.js`
- **LLM docs sync**: la documentación de `llm/` quedó alineada con `adapters/openai_adapter.py`
- **Health doc sync**: `docs/HEALTH.md` ya nombra `OpenAIAdapter` en la auditoría de salud
- **Error UI DOM**: la UI de error de stream dejó de reconstruirse con `innerHTML`
- **Tool UI DOM**: las pills de tool calls ahora se renderizan con nodos DOM explícitos
- **Reasoning UI DOM**: el bloque inicial de razonamiento ahora se construye con nodos DOM explícitos
- **Content render seam**: el render principal de `content-handler.js` quedó detrás de `setSegmentContent()`
- **Markdown render seam**: `markdown-renderer.js` pasó a usar `setRenderedHtml()` en `renderAll()`
- **Markdown fallback cleanup**: el fallback de `markdown-renderer.js` quedó centralizado en `fallbackSetHtml()`
- **Sidebar render seam**: `sidebar-refresh.js` pasó a usar `replaceChildren()` con `template.content`
- **Tool persister explícito**: `_persist_tool_results()` ya no resuelve repositorios por su cuenta y exige `repos` explícito
- **Tool history explícito**: `get_tool_history` dejó de resolver `get_repos()` internamente y exige `_repos` explícito
- **Message render explícito**: `render_session_messages()` y la ruta `/sessions/{session_id}/messages` recibieron `repos` explícito; el home/session page dejó de resolver modelos desde discovery
- **Frontend de transición final**: `session-page.js`, `chat-form.js`, `retry-handler.js` y `widgets/toolbar-editor.js` dejaron de depender de `innerHTML` directo en sus rutas críticas
