# v0.0.54 — 2026-06-13

## Lego Hardening de Backend y Transición Frontend

### Backend / Core
- **Runtime sin wrappers viejos**: eliminados `src/api/llm.py`, `src/api/models.py`, `src/api/history.py` y `src/api/health.py`
- **`save_message()` legacy fuera del runtime**: `src/api/messages.py` quedó con `save_message_record()` como contrato canónico
- **Historial tipado**: `HistoryMessage` estabiliza el contrato entre parser, UI y reconstrucción
- **`rebuild_history()` explícito**: ahora exige `messages_repo` en vez de caer a `get_repos()` por defecto
- **Sesión más limpia**: la cascada de borrado se movió a `SessionRepository.delete_cascade()`
- **Contrato de sesión afinado**: `conn_fn` salió del contrato público de sesión

### Contexto / Tools
- **`load_context()` puro**: ya no escribe `TOOLS.md` ni archivos de reglas durante la lectura
- **Snapshot explícito**: la materialización de contexto quedó separada del armado del prompt
- **Loader más predecible**: `src.tools` ya no dispara el auto-build al importar el paquete

### Web / Frontend
- **Imports directos**: routers y servicios web dejaron de depender de wrappers obsoletos
- **Transición acotada**: la compatibilidad legacy quedó concentrada en las superficies que todavía la necesitan
- **Historia compartida más estable**: el frontend ya consume el contrato de historial normalizado

### Documentación
- **ROADMAP.md**: actualizado con el estado real del refactor Lego
- **docs/LEGOS_AUDIT.md**: backlog y resolución alineados con backend y frontend actuales

### Verificación
- Verificación manual de estado del worktree y alineación documental
