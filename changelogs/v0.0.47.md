# v0.0.47 — 2026-06-12

## Cambios

- `src.core.orchestrator` ahora recibe el registry de tools por parámetro y deja de depender de la fachada `src.tools` como punto de acople.
- `src.core.tool_loop` ya no usa un singleton de `MessageRepository`; el repo se inyecta y, si no viene, se crea localmente en el borde.
- `src.core.history_rebuilder` aceptan `MessageRepository` inyectable, y `web/routers/chat.py` lo pasa de forma explícita desde `get_repos()`.
- La ruta activa conserva el comportamiento, pero el núcleo queda más fácil de aislar en tests y refactors futuros.
