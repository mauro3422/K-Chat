# v0.0.26 — Repository registry removal and facade cleanup

## Bug Fixes

### Repository access
- Removed the shared `src.api._repos._get_repo()` cache layer from runtime code.
- `src/api/messages.py`, `src/api/session.py`, `src/api/tools.py`, `src/api/widgets.py`, `src/api/debug.py`, `src/background_tasks.py`, `src/core/history_rebuilder.py`, `src/tools/_tool_persister.py`, `src/tools/_widget_helpers.py`, `src/tools/get_tool_history.py`, and `src/tools/runner.py` now use module-level repository singletons directly.
- `src/api/_repos.py` was deleted after the last runtime callers moved off it.

### Tests and contracts
- Updated the unit tests that still patched the old registry to patch the module-level singleton owned by each seam.
- Fixed the `chat()` test seam to patch `src.core.chat_sync.llm_chat`, which is the real import boundary for the sync wrapper.
- Updated the architecture docs so the module map and lazy-singleton notes match the current runtime shape.
