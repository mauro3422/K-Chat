# v0.0.25 — Contract hardening, bootstrap split & facade cleanup

## Bug Fixes

### Web startup
- `web/routers/pages.py`: model list generation now uses cached verified IDs and local fallback instead of running verification during request handling.
- `tests/integration/test_web_server.py`: switched from `TestClient` lifecycle coupling to direct handler assertions, so the web suite no longer hangs on startup behavior.

### Stream contract
- Added shared NDJSON stream contract modules:
  - `web/services/stream_contract.py`
  - `web/static/modules/stream-contract.js`
- `web/services/chat_stream.py` now serializes all stream events through the shared contract, including recovery-path events.
- `web/static/modules/stream-fetcher.js` now validates incoming NDJSON with the shared parser.

### Widget and stream bootstraps
- Split `window.KairosWidgets`, `window.StreamOrchestrator`, and `window.KairosForm` compatibility into dedicated bootstrap modules.
- Kept the real widget/form/stream modules export-only, so the compatibility layer no longer lives inside the core modules.

### API and dependency boundaries
- `src/api/__init__.py` was trimmed further as a compatibility-only facade.
- `src/core/_deps.py` moved from `functools.partial` wiring to explicit wrapper functions around the LLM client, keeping the injection point visible.
- `src/core/tool_loop.py` now imports `save_message` locally where it is used, avoiding a core import cycle while fixing the sync save path.

## Tests

- Python: `tests/unit/test_pages_router.py`, `tests/integration/test_web_server.py`
- JS: `tests/test-frontend-integration.test.js`, `tests/test-chat-stream.test.js`, `tests/test-widget-system.test.js`, `tests/test-stream-contract.test.js`

## Notes

- This release continues the "lego" work: fewer hidden entry points, clearer compatibility shims, and stricter contracts between web, core, and frontend layers.
