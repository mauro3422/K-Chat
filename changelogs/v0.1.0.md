# v0.1.0 — 2026-06-20

32 commits desde v0.0.64. Refactor interno masivo (Fase 3 de DI), federación LAN, y overhaul de frontend.

## LAN Sync / Federación
- Sincronización de sesiones entre nodos LAN vía `lan_bridge.py` con leader lease distribuido
- Failover automático de nodo líder con `leader_lease.py` y `memory_lease.py`
- `src/coordination/`: node_state, leader_lease, memory_lease, memory_write_queue, lan_bridge
- `node.py` router: peers, health checks, acciones proxy sobre nodos remotos
- Federación de directorio de sesiones entre nodos (`web/services/session_directory.py`)
- Panel "Estado LAN" en vivo (HealthOverviewPanel, LanStatusPanel, MemoryStatusPanel widgets)
- Diagnóstico unificado en `/diagnostics` con tabla de peers, memoria remota, drift, diff por sesión
- Página interactiva de diagnósticos (`diagnostics.html`) con links a peers y acciones inline
- Peer memory: comparación local vs remota, detección de drift, explicación de causas
- Sesión master con entrypoint dedicado y link en UI
- Queue de escritura coordinada para evitar conflictos entre nodos
- Estado de failover trackeado en `web/services/failover_state.py`
- Check-list de validación two-PC, runbook operativo, status report

## DI Composition Root (Fase 3)
- Servicios core migrados al composition root en `web/app_factory.py` (6 commits)
- Todos los singletons reubicados a `app.state` — eliminados imports globales de `DEFAULT_CONFIG`
- `src/llm/container.py`: refactor completo, singletons creados desde composition root
- LLM container movido a `app.state` con lazy init desde `load_config()`; fallbacks eliminados
- Connection pool de base de datos (`src/memory/connection_pool.py`) gestionado desde composition root
- Curator config inyectado vía DI en lugar de import directo
- Tests de regresión y `test_anti_regression.py` actualizados para nuevo patrón DI

## Multi-device SSE
- `KAIROS_WEB_URL` acepta múltiples URLs separadas por coma
- `adapter.py`: `_get_sse_notify_urls()` devuelve lista; `_notify_all()` POSTea a todas
- `ws_client.py`: fallback HTTP usa `_notify_all()` en vez de notificar a una sola URL
- Cada error de notificación se loggea sin bloquear al resto de los dispositivos

## Telegram Channel
- `telegram/adapter.py`: refactor de eventos WebSocket y manejo de reconexión
- `telegram/ws_client.py`: notificaciones multi-URL vía `_notify_all()` en fallback HTTP
- `web/services/telegram_reflection.py`: reflexión de mensajes Telegram en Web
- Nuevos tests de integración: `test_telegram_bot.py`, `test_telegram_reflection.py`, `test_adapter_ws_events.py`

## Async Tools & Lifecycle
- Runtime async estabilizado: herramientas ejecutan con manejo de concurrencia consistente
- Flujo de sesión refinado: ciclo de vida predecible entre orchestrator, tool_loop y runner
- `async_utils.py` expandido con helpers de scheduling y cancelación

## Frontend
- Selector de modelos custom con SVGs de capacidades (razonamiento, tools, imagen, video, audio)
- Indicadores de estado por tier (GO Premium/Standard/Económico, Free) con dots de color SVG
- Dropdown inteligente con detección de espacio (abre hacia arriba si no hay lugar abajo)
- Gutter drag funcional para redimensionar sidebar (mouse + touch, 160-500px)
- SkillsUI movido a dropdown tipo notificaciones desde el sidebar
- 21 SVGs oficiales guardados en `web/static/icons/`
- Panel de estado (LAN, memoria, salud) movido a solapa "Estado" en Debug
- Sección "Session Data" en Debug con JSON completo de sesión (auto-load)
- Manejo de errores mejorado: detección de quota/auth/rate_limit con tarjetas específicas
- Persistencia del ancho del sidebar en localStorage

- Scroll reseteado en panel de Debug por regeneración HTML
- Import incorrecto `ILogger` en SystemLogPanel

## Limpieza
