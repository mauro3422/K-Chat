# v0.1.0 — 2026-06-08

Primera versión estable de Kairos. Agente personal con chat streaming, herramientas paralelas y dashboard web.

## Novedades

### Núcleo
- `config.py`: lectura de .env con validación de OPENCODE_ZEN_API_KEY
- `src/llm.py`: cliente OpenAI-compatible para OpenCode Zen API, detección automática de modelos free
- `src/memory.py`: SQLite con sesiones, mensajes (razonamiento + fases), tool_calls (con turno), debug_info
- `src/core.py`: orquestador con tool loop, ejecución paralela (ThreadPoolExecutor), history compressor

### Tools
- Auto-registro via importlib — cada archivo en `src/tools/` exporta DEFINITION + run()
- `web_search`: búsqueda DuckDuckGo
- `get_tool_history`: el modelo consulta su propio historial de herramientas
- Tools válidas: se valida DEFINITION + run() al cargar

### Dashboard web
- FastAPI + HTMX + Jinja2 (sin build step, sin npm)
- Streaming NDJSON con tipos: reasoning, content, tool_call
- Razonamiento visible en `<details>` por fase
- Tool pills con spinner (calling) y checkmark (ok)
- Sidebar con sesiones (rename, delete, navegación)
- Panel Debug lateral: modelo, razonamiento, tools, stream log, UI log

### Persistencia visual post-F5
- `turn` en tool_calls: cada iteración del tool loop se numera
- `phases` JSON en messages: razonamiento + tool_ids por fase
- Histórico intercala razonamiento ↔ tools ↔ contenido al recargar
- `html.escape()` en todo para prevenir HTML injection del modelo
- Cache headers desactivados en rutas HTML

### Calidad
- 23 tests unitarios (memory CRUD, tool loop, streaming, session_id)
- WAL mode + busy timeout en SQLite para concurrencia
- `try/finally` en todas las conexiones SQLite (sin leaks)
- `except:` estrechado a excepciones específicas (`OperationalError`, `JSONDecodeError`)
- Logging (logger.info) reemplaza prints en core.py
- `SESSION_ID` propagado como kwarg a tools (sin globales)
- `USERNAME` + `USER` cross-platform (Windows/Linux)
- `.gitignore` completo (secrets, python, db, tests, ide, os)

### Documentación
- README.md completo con estructura, funcionalidades, arranque, tests
- ROADMAP.md con fases completadas y próximas
- docs/ARCHITECTURE.md con filosofía legos, flujo de datos, wire format

### Infraestructura
- `.env.example` limpio (sin valores reales)
- requirements.txt con dependencias
