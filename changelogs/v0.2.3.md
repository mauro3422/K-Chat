# v0.2.3 - 2026-06-29

## Refactor

### `web/routers/node.py` god-router split

`node.py` era el router más grande del proyecto (519 líneas, 14 endpoints, ~8 helpers privados) mezclando 3 dominios: coordinación de nodo, memoria distribuida, y failover. Spliteado en 6 archivos cohesivos:

- `web/routers/node.py` (202 líneas) — coordinación: state, runtime, heartbeat, promote, demote, event, sessions
- `web/routers/node_memory.py` (184 líneas) — memoria distribuida: memory/request, memory/queue, memory/flush, sync/status, memory/snapshot, diagnostics
- `web/routers/node_failover.py` (23 líneas) — failover/status
- `web/routers/_node_helpers.py` (96 líneas) — 9 `_get_*` providers + `_peer_cluster_state` + wrapper `_request_repos`
- `web/routers/_node_models.py` (27 líneas) — 3 Pydantic models vivos
- `web/services/node_observability.py` (95 líneas) — 5 helpers de observability (pura lógica, sin imports FastAPI)

URLs IDÉNTICAS — las 14 rutas `/api/node/*` se registran sin cambios via auto-discovery (todos los routers comparten el prefix `/api/node`). Sin breaking changes.

**Decisión**: observability helpers fueron a `web/services/` (no a `web/routers/`) porque son lógica de dominio pura — sin `Request`, sin `APIRouter`. Cumple "framework concerns belong in web/" en sentido inverso.

### Providers deduplication

Antes del refactor, `_get_coordinator`, `_get_event_bus` y el wrapper `_request_repos` estaban duplicados en 3-4 módulos cada uno (`memory.py`, `_memory_snapshot.py`, `sessions.py`, `pages.py`, `node.py`). Ahora se importan de `_node_helpers.py` desde todos los routers. Sin duplicación.

**No consolidados** (semántica distinta por diseño, documentado en `docs/REFACTOR_PENDING.md`):
- `_get_node_bridge` de `health.py` (devuelve `None` si no hay coordinator para graceful degradation)
- `_request_bridge` de `sessions.py` (valida mocks con `is_unconfigured_mock`)
- `_request_base_url` de `sessions.py` (None-tolerant)
- `_request_web_base_url` de `pages.py` (prioriza `config.web_base_url`)
- `_get_lease_manager` de `memory.py` (usa MEMORY_lease != LEADER_lease)

### `NodeRolePayload` eliminado — dead code desde commit `feac04d`

Investigación con `git pickaxe`: el model `NodeRolePayload` se creó en `feac04d` (commit original de LAN sync) en el MISMO momento en que `/promote` y `/demote` se definieron sin body payload (`async def node_promote(request: Request)`). Nunca existió un endpoint que lo recibiera, ni un test, ni un caller. Dead code de nacimiento. Eliminado.

### TS migration — cierre cosmético completado

La migración TS estaba más completa de lo que parecía. El source TS era el frontend real; el JS vanilla de aplicación ya no existía. Lo que faltaba era cierre cosmético:

1. **`app_mock.ts` → `app.ts`** (vía `git mv`). `vite.config.js`, `pages.py`, tests y docs actualizados. Bundle regenerado como `dist/assets/app.js` (42 KB gzip 12.4). 0 refs a `app_mock` en source/tests/docs.

2. **`model-availability.js` migrado a TS** como `web/src_ts/widgets/ModelAvailabilityPoller.ts` (~110 líneas). Preserva: polling `/models/availability` cada 60s, `rl-badge` junto a `#model-select-wrapper`, banner `go-quota-warning`. Elimina: manipulación del `<select>` invisible (dead code ahora que `ModelSelector.ts` usa dropdown custom en divs). Wireado en `app.ts` como bloque DI. Archivo JS vanilla y `<script>` del template eliminados.

3. **`web/routers/chat_ts.py` eliminado** (vía `git rm`). La URL `/chat-ts` ya no existe; las rutas principales `/`, `/session/{id}` sirven directamente el template TS.
