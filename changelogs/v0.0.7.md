# v0.0.7 — 2026-06-11

## Agregado

### Arquitectura — Refactor mayor
- **`src/context.py` → `src/context/` package**: dividido en 4 módulos (templates.py, files.py, tools_docs.py, builder.py) con shim de retrocompatibilidad
- **`src/core/history.py` → 3 módulos**: history_parser.py, history_rebuilder.py, history_ui.py con shim
- **`src/tools/runner.py` → 4 módulos**: _rate_limiter.py, _tool_parser.py, _tool_persister.py + runner.py como orquestador
- **`src/memory/database.py`**: DatabaseEngine Protocol + SQLiteEngine (abstracción de engine para futuro soporte multi-DB)
- **`src/llm/models.py`**: Provider injection con `_PROVIDER_REGISTRY` + `register_provider()` + config por env `LLM_PROVIDER`
- **`web/services/chat_stream.py`**: extraídos stream_error_classifier.py y message_persister.py

### Ciclo de dependencias roto
- **tools↔api**: 4 tools (save_widget, update_widget, get_widget_code, get_tool_history) ahora importan directo de repos en vez de src.api
- **toolbar↔iframe**: dependencia circular rota via iframe-builder.js + ui-helpers.js

### Frontend — Desacoplamiento
- **`stream-renderer.js`** (258→10 líneas): dividido en 3 handlers independientes de streaming
- **`chat-form.js`**: extraídos los handlers de retry y error, eliminado monkey-patching de KairosStream.emit
- **`session.js`**: envuelto en IIFE `KairosSession` con aliases backwards-compatibles

### CLI
- `cli.py` y `handler_cli.py` ahora usan `src.api` como fachada (eliminados imports directos de core/memory)

### api.py
- 6 singletons de repos → 1 lazy registry `_get_repo(cls, name)` (~40 líneas menos de boilerplate)
