# v0.0.23 — Widget rendering, stream stability & logging

## Bug Fixes

### Widget rendering
- Fixed double-boxing: `.msg.assistant` now hides background/border when containing a widget
- Fixed iframe height: changed from fixed 200px to `height: 0` + `min-height: 60px` (grows with content)
- Fixed `scrolling="no"` → removed, iframe now uses `overflow: hidden` with auto-resize via postMessage
- Fixed placeholder height: from fixed 200px to `min-height: 80px`
- Fixed toolbar: added `flexWrap` and `gap` for narrow screens
- Fixed widget width: messages with widgets now use `max-width: 95%` instead of 80%

### Widget deduplication
- `content-handler.js`: widgets are now deduplicated by key — if an `html-widget` block already created a widget with key X, the `[Widget: X]` tag is skipped
- `content-handler.js`: tags inside code blocks or text references no longer create widget containers

### Stream stability
- Fixed `fetchVersionLabel` 404 noise: toolbar now skips API fetch if widget code was loaded from cache
- Fixed frontend timeout not clearing after stream completes: added `clearTimeout(timeoutId)` after successful fetch
- Fixed history sanitizer: assistant messages without `tool_calls` that appear between a tool_calls assistant and its tool responses are now skipped (prevents `400 Bad Request` on retry)
- Fixed `message_renderer.py`: now injects `_code_{key}` entries into `data-widget-states` so inline widgets persist across page reloads
- Fixed `chat-stream.js`: loads `_code_` entries from `data-widget-states` on initial page load (was only loaded in `loadSession`)
- Fixed `message_repository.py`: increased message limit from 200 to 500 (was cutting off recent widgets)

### Loop detection
- Added `_LoopDetector` in `chat_stream.py`: detects repeated tokens (10x same token) and repeated phrases (3x same 50-200 char phrase)
- Reports `loop_detected` error type to frontend when loop is detected

### Backend logging
- Fixed `web/logging.py`: `BackendLogHandler` now installs on root logger instead of `kairos` logger (app loggers use `__name__` which aren't children of `kairos`)
- Added `logging.basicConfig()` in `server.py` with configurable `LOG_LEVEL` env var
- Added periodic save every 30s during stream to prevent data loss on interruption

### SearXNG
- Fixed `dependencies/manage.py`: added `--break-system-packages` flag for pip install (required for Python 3.14 externally-managed environments)

## Changes

### Skills & Prompts (English)
- `skills/html-widgets.md`: translated to English, added mandatory size limits (max 600px height, no scroll, no fixed height)
- `skills/INDEX.md`: translated to English
- `AGENTS.md`: fixed Spanish fragments (`opcional`, `ej:`, `URL completa`, `Mostreale`, `Día`, `contador:`)

### Frontend timeout
- `retry-handler.js`: increased stream timeout from 30s to 120s

## Tests

- Updated `test_web_logging.py`: checks root logger instead of `kairos` logger
